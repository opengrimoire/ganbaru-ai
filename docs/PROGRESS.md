## What exists (phase 1)

- Monorepo scaffold (Turborepo + pnpm workspaces + Cargo workspace)
- Tauri v2 + Svelte 5 + Vite app shell with tab navigation
- shadcn-svelte + Tailwind CSS v4, custom neutral gray theme (dark + light), custom title bar
- shared-types package
- SQLite schema with migrations (calendar_events, pomodoro_sessions, pomodoro_configs, pomodoro_segments, tasks, calendars, attendees, alarms, overrides)
- Calendar: custom Google Calendar-style component (week/day/month views, drag-and-drop create/move/resize, recurring events with full RFC 5545 RRULE support, all-day events, iCalendar field parity, edit session overlay model, EventPanel with 8 extracted sub-components including unified Meeting section that bundles attendees, URL, and location under one collapsible, 275 unit tests)
- Kanban board (backlog/todo/in_progress/done columns, CRUD, move between columns, priority badges)
- Pomodoro timer (focus/break phases, cycle tracking, segment lifecycle, timeline rail visualization, idle detection, suspend/wake handling)
- Theming: registry-based theme store with two built-in themes plus full user-theme CRUD (create, duplicate, edit, rename, delete, plus `replaceTheme` for in-place JSON edits). Layered token model: a seven-color `sources` palette (canvas, ink, primary, destructive, confirm, warning, calCanvas) feeds a derivation engine that tints the rest of the shell automatically. Beyond the canvas/ink/primary/destructive foundation, `confirm` and `warning` act as semantic-signal sources (identity-driving the save button and active-scope pill plus the accepted attendance tile from confirm, and the tentative attendance tile from warning); destructive likewise identity-drives the armed-delete button and declined attendance tile; ink lifts drive the form-indicator dot, pomodoro idle caption, and event-panel dim captions. A three-layer resolver (override, derived, base CSS) keeps Quick colors cohesive while letting users isolate individual tokens. Clone always synthesizes sources from the resolved palette so every new user theme opens in Quick-colors mode, with explicit overrides preserved as isolated tokens. `cloneTheme` also captures an input-seed snapshot (`seedSources`, `seedAppTokenOverrides`, `seedCalendarTokenOverrides`, `seedBlendCanvas`, `seedEventPalette`) that powers per-row reset: every source color and every driven token has a persistent reset icon that enables when the current value diverges from the seed and clicking it restores just that one control (overrides revert to their seed hex, seed-less overrides are dropped so derivation takes over). Seeds persist in the vault but are stripped from exported JSON. Editor is organized as a three-tier walkthrough. Tier 1 (App foundation) covers App canvas, Ink (with sub-rows for foreground, form-indicator, pomodoro-idle-text), Primary action, and Destructive (with sub-rows for destructive, action-danger-armed, status-declined). Tier 2 (Semantic signals) covers Confirm (paired save button bg/fg plus accepted status) and Warning (tentative status). Tier 3 (Per-feature) covers Calendar canvas, Calendar details (merged markers + extras: today marker pair, now line, break and focus markers, color-picker outline, description editor tint, drag-preview border), Event panel (9 tokens), and Task priority (4 tokens). The collapse button is gated on (source + 2+ rows), so single-row source groups render their sub-row as a peer of the source, and sourceless cards (Calendar details, Event panel, Task priority) show every row inline. Legacy themes cloned before confirm and warning existed load without migration: missing source channels backfill from base defaults. `ColorField` accepts 8-digit `#rrggbbaa` so alpha-bearing tokens (panel edge/shadow, description editor tint, drag preview border) edit cleanly. Managed from a single Appearance settings page that lists every theme with per-row Apply, Duplicate and edit, View or Edit, and Delete actions above the font and zoom controls. Every open-for-edit path (New, Duplicate and edit, Edit) activates the edited theme on the spot and hands off to a floating, draggable editor panel: the Settings modal closes, the panel mounts without a backdrop so the app stays clickable, and a chevron in the header collapses the body so only the drag handle and footer stay visible (the X has been removed; collapse replaces it). The footer exposes Back to themes, which rolls the session back by restoring the pre-edit JSON via `replaceTheme`, deleting the theme outright when it was minted for this session, and reinstating the previously active theme; and Save and apply, which keeps the edits and returns to the list with the edited theme active (built-ins show Apply and return since there is nothing to save). A forced close through the title-bar close dialog triggers a best-effort cancel before quitting. Built-ins open in the same panel in read-only mode (palette preview and copy or save JSON panel); user themes show a quick-first editor built around a link metaphor. Each source color heads a card containing the tokens that derive from it. Multi-row groups are collapsible accordions starting collapsed, with an EXPAND / COLLAPSE button on the right that shares the fixed minimum width of the row-level Isolated edit / Link back buttons; single-row groups (Ink, Primary) drop the EXPAND / COLLAPSE button and render their one sub-row below the header with header-matching typography (bold 13px) and an always-editable `ColorField` (no Isolated edit / Link back button), so the driven token reads as a peer of its source; Destructive has no row at all because its source feeds `--destructive` through identity derivation with no other consumers. Multi-row groups keep two states per row: Linked (the `ColorField` renders disabled with reduced opacity and a not-allowed cursor, paired with an Isolated edit button) and Isolated (the same `ColorField` becomes editable, paired with a Link back button). The in-house HSL color picker drives every editable field and an editable JSON panel whose Apply validates and commits through `replaceTheme` (id locked). JSON import/export works through both clipboard and native file dialog (tauri-plugin-dialog).
- Vault foundation: `vault/config.json` on disk with atomic write (tmp + fsync + rename) via Rust commands `vault_read_config` / `vault_write_config`. Frontend bridge (`lib/vault/config.ts`) caches the config, exposes synchronous reads, and debounces writes. One-time migration moves the legacy localStorage keys (`ganbaruai-theme`, `ganbaruai-font-family`, `ganbaruai-font-scale`) into the vault. Active theme, user themes, font family, and font scale all persist through the vault now. Two extra commands `vault_read_text` / `vault_write_text` back the dialog-driven import/export of theme files.
- Settings modal (`SettingsModal.svelte`) opens from the title-bar gear icon and hosts a single Appearance section that combines the theme list, font family, and three steppers (text size, app zoom, calendar zoom). Popovers inside the modal (color picker, font family dropdown) portal under `<body>` and position themselves with `position: fixed` so they escape the modal's `overflow` ancestors. The theme editor itself lives in a separate floating panel (`FloatingThemeEditor.svelte`) mounted at the TitleBar level: a small `themeEditor.svelte.ts` store holds the editing theme id, an effect inside the Settings modal watches it and dismisses the modal whenever an editor opens so the app beneath stays clickable, and the TitleBar renders the panel whenever the id is set. While the editor is open the TitleBar disables the theme toggle, reset-database, help, and settings buttons so they can't disrupt the edit session; window controls, pomodoro, and the performance monitor stay live. Webview zoom lives in a single `zoom.svelte.ts` store; the calendar header zoom controls and the title-bar reset-zoom button moved into the modal (keyboard shortcuts preserved).
