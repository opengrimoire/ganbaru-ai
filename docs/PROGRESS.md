## What exists (phase 1)

- Monorepo scaffold (Turborepo + pnpm workspaces + Cargo workspace)
- Tauri v2 + Svelte 5 + Vite app shell with tab navigation
- shadcn-svelte + Tailwind CSS v4, custom neutral gray theme (dark + light), custom title bar
- shared-types package
- SQLite schema with migrations (calendar_events, pomodoro_sessions, pomodoro_configs, pomodoro_segments, tasks, calendars, attendees, alarms, overrides)
- Calendar: custom Google Calendar-style component (week/day/month views, drag-and-drop create/move/resize, recurring events with full RFC 5545 RRULE support, all-day events, iCalendar field parity, edit session overlay model, EventPanel with 8 extracted sub-components including unified Meeting section that bundles attendees, URL, and location under one collapsible, 275 unit tests)
- Kanban board (backlog/todo/in_progress/done columns, CRUD, move between columns, priority badges)
- Pomodoro timer (focus/break phases, cycle tracking, segment lifecycle, timeline rail visualization, idle detection, suspend/wake handling)
- Theming: registry-based theme store with two built-in themes plus full user-theme CRUD (create, duplicate, edit, rename, delete, plus `replaceTheme` for in-place JSON edits). Layered token model: a seven-color `sources` palette (canvas, ink, primary, destructive, confirm, warning, calCanvas) feeds a derivation engine that tints the rest of the shell automatically. Surface backgrounds still use empirical sRGB lifts for the tinted-gray aesthetic, but every foreground, border, and muted caption is recomputed from WCAG contrast math in `components/ui/colorMath.ts` (relative luminance, contrast ratio, OKLab lightness walks) through `pickReadableForeground` (AA 4.5:1 body text), `pickReadableBorder` (3:1), and `pickReadableMuted` (parked at exactly AA-large so captions recede without vanishing). Six new tokens join the shell to make every hardcoded `text-white` contrast-pickable: `--destructive-foreground`, `--action-danger-armed-foreground`, `--status-accepted-foreground`, `--status-tentative-foreground`, `--status-declined-foreground`, and `--pomodoro-idle-timer`. Hardcoded foregrounds on buttons, badges, title bar close, event panel, meeting statuses, kanban priority, and calendar month-view day numbers all now consume the token-driven or contrast-picked equivalents; the light-overlay pomodoro timer flips automatically when canvas/ink flip. A three-layer resolver (override, derived, base CSS) keeps Quick colors cohesive while letting users isolate individual tokens (50 app tokens, 10 calendar tokens). `--foreground` itself is contrast-picked against canvas now (no longer identity-equal to ink), so darkening the canvas automatically flips body text everywhere instead of stranding black text on a dark background. A sibling `--card-foreground` token derives the same way against the card surface (kept pure white in light mode) so popovers and dialogs painted on cards stay legible even when canvas and ink diverge; every in-app `bg-card` wrapper aliases `--foreground` to `--card-foreground` inside its scope so inner children that still reference `text-foreground` or inline `var(--foreground)` pick up the right color without per-class rewrites. Clone always synthesizes sources from the resolved palette so every new user theme opens in Quick-colors mode, with explicit overrides preserved as isolated tokens. `cloneTheme` also captures an input-seed snapshot (`seedSources`, `seedAppTokenOverrides`, `seedCalendarTokenOverrides`, `seedBlendCanvas`, `seedEventPalette`) that powers per-row reset: every source color and every driven token has a persistent reset icon that enables when the current value diverges from the seed and clicking it restores just that one control. The editor footer exposes "Reset all" which restores every source, override, palette slot, and blend canvas from seeds in one update. Legacy themes loaded from disk without seeds run through `synthesizeSeedsIfMissing` on load (idempotent, sampled from sources or resolved tokens) so reset works uniformly. Seeds persist in the vault but are stripped from exported JSON. Managed from a single Appearance settings page that lists every theme with per-row Apply, Duplicate and edit, View or Edit, and Delete actions above the font and zoom controls. Every open-for-edit path (New, Duplicate and edit, Edit) activates the edited theme on the spot and hands off to a floating, draggable editor panel; the Settings modal closes, the panel mounts without a backdrop so the app stays clickable, and a chevron in the header collapses the body so only the drag handle and footer stay visible. The footer exposes Back to themes (rollback or delete session-minted clone, plus reinstate prior active), Reset all, and Save and apply; a forced close through the title-bar close dialog triggers a best-effort cancel. Clicking "New theme" opens a preset picker before the editor: six curated themes (Sunrise, Graphite, Sepia, Nordic, High-contrast Dark, Lavender Pastel) render as miniature previews painted from their own sources, each build-time validated to meet AA contrast across every derived token; "Start blank" keeps the original seed-from-active path. The editor body itself is gated by a three-segment mode selector. Essentials shows only the four foundation sources; Accents adds confirm, warning, and calendar canvas; Advanced surfaces the full three-tier walkthrough with every driven sub-row reachable for isolation. Defaults to Essentials on fresh clones, Advanced on themes that already carry overrides. A live preview pane (`ThemePreviewPane`) renders the theme's actual tokens (re-declared inline so the preview paints from the theme, not the editor chrome) across button rows, priority badges, text samples, and a 3-day calendar mini. Every pair row shows a live contrast indicator: when the pair falls below AA 4.5:1 the editor surfaces an amber ratio chip and a wand auto-fix button that calls `pickReadableForeground` and writes the result as an override. A persistent contrast-summary bar above the preview reports the total number of failing pairs across every mode and collapsed group, with a "Jump to next" button that cycles through failing rows (auto-switching to Advanced and expanding the enclosing group as needed) and a "Fix all" button that runs the per-row auto-fix on every failing pair at once, so warnings produced by a distant edit cannot hide behind an unopened accordion. Editor chrome is decoupled from user tokens: `.theme-editor-chrome` shadows the user-facing CSS variables inside the editor scope to stable `--editor-chrome-*` values, so the feedback loop never self-destructs (the editor stays legible even when canvas equals ink). The scope travels with the `ColorField` popover via its portaled wrapper; slider thumbs read `--editor-chrome-thumb-border`; the alpha checker uses a conic-gradient driven by `--editor-chrome-checker-a` / `-b`. `ColorField` accepts 8-digit `#rrggbbaa` so alpha-bearing tokens (panel edge/shadow, description editor tint, drag preview border) edit cleanly. Built-ins open in the same panel in read-only mode (palette preview and copy or save JSON panel). The in-house HSL color picker drives every editable field and an editable JSON panel whose Apply validates and commits through `replaceTheme` (id locked). JSON import/export works through both clipboard and native file dialog (tauri-plugin-dialog).
- Vault foundation: `vault/config.json` on disk with atomic write (tmp + fsync + rename) via Rust commands `vault_read_config` / `vault_write_config`. Frontend bridge (`lib/vault/config.ts`) caches the config, exposes synchronous reads, and debounces writes. One-time migration moves the legacy localStorage keys (`ganbaruai-theme`, `ganbaruai-font-family`, `ganbaruai-font-scale`) into the vault. Active theme, user themes, font family, and font scale all persist through the vault now. Two extra commands `vault_read_text` / `vault_write_text` back the dialog-driven import/export of theme files.
- Settings modal (`SettingsModal.svelte`) opens from the title-bar gear icon and hosts a single Appearance section that combines the theme list, font family, and three steppers (text size, app zoom, calendar zoom). Popovers inside the modal (color picker, font family dropdown) portal under `<body>` and position themselves with `position: fixed` so they escape the modal's `overflow` ancestors. The theme editor itself lives in a separate floating panel (`FloatingThemeEditor.svelte`) mounted at the TitleBar level: a small `themeEditor.svelte.ts` store holds the editing theme id, an effect inside the Settings modal watches it and dismisses the modal whenever an editor opens so the app beneath stays clickable, and the TitleBar renders the panel whenever the id is set. While the editor is open the TitleBar disables the theme toggle, reset-database, help, and settings buttons so they can't disrupt the edit session; window controls, pomodoro, and the performance monitor stay live. Webview zoom lives in a single `zoom.svelte.ts` store; the calendar header zoom controls and the title-bar reset-zoom button moved into the modal (keyboard shortcuts preserved).
