Timezone picker is Google-Calendar-style with a three-button label-format toggle (`Acronym` / `UTC only` / `UTC fallback`) in the popover header that rewrites every trigger column label consistently: in `Acronym` mode the trigger shows the Intl-provided short code (`PST`, `CST`, etc.) when CLDR exposes one and otherwise a derived 2-5 letter acronym from the long name (`KST`, `JST`, `AEST`, `CEST`, `NZST`, `WAT`, `HAST`, `ACWST`, `FNST`, `WFT`, splitting on whitespace, hyphens, and ampersands and skipping lowercase particles like `de` so `Fernando de Noronha Standard Time` yields `FNST` and `Wallis & Futuna Time` yields `WFT`, falling back to the stripped numeric form `+14` / `-11` only for the 13 zones whose long name itself is offset-shaped, e.g. `Asia/Amman`, `Pacific/Bougainville`); `UTC only` mode shows the compact numeric form for every row (`-6`, `+9`, `+5:30`, `+14`) regardless of whether Intl exposes a short, since `UTC-08:00` / `GMT+5:30` overflow the 46px column at 13px and so users opting in get a fully consistent numeric column; `UTC fallback` mode mirrors what Google Calendar does today (Intl short when available, numeric otherwise, leaving a mix of `CST` and `+9`). The popover offset always renders as `UTC±HH:MM` in all three modes, the portal-anchored popover browses every IANA zone (with `Etc/*` and deprecated single-segment aliases filtered) and supports ranked multi-field search across Intl long name, city, region, abbrev, and IANA id with prefix-over-contains tiering, picker rows show two lines, top line `offset Long Name` both foreground (offset font-medium for slight emphasis), bottom line `City, Region` in a smaller muted font; the offset is pinned to a fixed-width tabular-nums span so every long name starts at the same x position regardless of offset length, active rows render the offset (also fixed-width tabular-nums) followed by the Intl long name and a `(device)` tag where applicable so the long name aligns vertically across rows (the column-form abbrev already lives in the trigger above so no need to repeat it), the active list lets the user reorder any row (including the device row) via HTML5 drag-and-drop on the entire row (chevrons and X are `draggable=false` so they don't initiate drags) or up/down chevrons (the device row keeps the `(device)` tag wherever it lands and its X is hidden so it can never be removed), up to five timezones can be active at once, the result list hides the native scrollbar and renders a custom themed thumb that overlays the right edge; the entire track is the pointer target, so clicking the empty track jumps the thumb to the click point and continues dragging from there, and body text selection is locked while a scrollbar drag is active so accidental pointer movement over rows can't start a selection, the popover `stopPropagation`s wheel events so the sticky header's wheel-nav handler can't hijack scrolling, arrow-key/Enter typeahead navigation is intercepted in capture phase so it never trips the calendar's window-level keyboard nav, and per-zone Intl metadata (offset, long name, city, region, abbrev, lowercase search mirrors) is baked once into a `getTimezoneInfo` cache that prewarms via `requestIdleCallback` on mount so first open is instant.

- Theming: registry-based theme store with two built-in themes plus full user-theme CRUD (create, duplicate, edit, rename, delete, plus `replaceTheme` for in-place JSON edits). User themes persist as a normalized SQLite snapshot across `themes`, `theme_tokens`, `theme_event_palette`, `theme_seed_tokens`, `theme_seed_event_palette`, and `theme_upgrade_dismissals`. Each theme stores a six-color `sources` palette (canvas, ink, primary, destructive, confirm, warning), a full resolved-token snapshot for every app and calendar shell variable, a per-token `isolated` flag, a `derivation_engine_version` stamp, plus seed mirrors of every column for per-row and "Reset all" reset. Built-in light and dark stay code-pinned (the schema's `CHECK (id NOT IN ('light', 'dark'))` defends against any import shadowing them). The boot sequence in `main.ts` awaits `ensureConfigLoaded()` and then `hydrateUserThemes()`, which runs an idempotent one-time migration that walks the legacy `themes.user` blob from `vault/config.json` through `validateThemeJson`, runs the current derivation engine to produce missing tokens, writes one transaction per theme, then strips `themes.user` from the config. JSON serves only as import/export interchange: v1 exports carry `schemaVersion: 1` plus the full snapshot, while legacy imports re-derive at import time and map the old `appTokenOverrides` / `calendarTokenOverrides` into the new isolated-flag sets. The derivation engine itself is unchanged: it is calibrated end-to-end against the dark built-in so feeding its sources through `deriveAppTokens` / `deriveCalendarTokens` reproduces `BASE_APP_TOKENS.dark` and `BASE_CALENDAR_TOKENS.dark` within a 2 rgb-unit tolerance, every formula is a signed OKLab ΔL delta off canvas or a `walkFraction(anchor, paired-bg, fraction)` between a per-surface `pickReadableForeground` anchor and that surface, calendar canvas is auto-derived from `canvas` via a direction-aware ΔL offset (-0.089 on dark, +0.032 on light) and stays pinnable through the `--cal-bg` isolated flag, and foregrounds split across `pickReadableForeground` (AA 4.5:1 on body-text pairs), `pickBrightForeground` (status/destructive/sidebar signal colors), a handful of hardcoded tokens (`--status-tentative-foreground` at `#FFFFFF`, `--form-indicator` at ink), and a calibrated OKLab-L shift on `--action-confirm-foreground`. The editor follows a buffer model: every mutator (source edit, isolate flip, link back, palette slot, rename, rebake, preset apply, per-row reset, "Reset all") updates the in-memory `UserTheme` in `$state` and re-paints the DOM but never touches SQLite during the session. Save and apply flushes the buffered theme through `persistThemeToDb`, which calls `insertTheme` for a fresh theme (created or duplicated this session) or `replaceThemeContent` for an existing one (UPDATE parent, DELETE+INSERT children, dismissals preserved). Back to themes drops the buffer: a fresh theme is removed entirely from the registry, an existing theme rolls back to the JSON snapshot captured when the editor opened. Editing a source value rebuilds the snapshot through `deriveAppTokens` and `deriveCalendarTokens`, leaves isolated tokens alone, and updates `blendCanvas` whenever `--cal-bg` participates so the dimmed event-tile blend stays in sync, all in memory until commit. `Isolated edit` flips the flag without touching the value, `Link back` re-runs derivation for that key and unflips, and "Reset all" restores every source, isolated flag, snapshot, palette slot, and blend canvas from the seed mirrors in one in-memory pass. Six tokens live in the shell to make every hardcoded `text-white` contrast-pickable: `--destructive-foreground`, `--action-danger-armed-foreground`, `--status-accepted-foreground`, `--status-tentative-foreground`, `--status-declined-foreground`, and `--pomodoro-idle-timer`; hardcoded foregrounds on buttons, badges, title bar close, event panel, meeting statuses, kanban priority, and calendar month-view day numbers all consume the token-driven or contrast-picked equivalents. `--foreground` is contrast-picked against canvas (no longer identity-equal to ink) and `--card-foreground` derives the same way against the card surface so popovers stay legible even when canvas and ink diverge. Managed from a single Appearance settings page that lists every theme with per-row Apply, Duplicate and edit, View or Edit, and Delete actions above the font and zoom controls. Every open-for-edit path activates the edited theme and hands off to a floating, draggable editor panel: the Settings modal closes, the panel mounts without a backdrop, a chevron in the header collapses the body, and the footer exposes Back to themes (rollback through `themeEditor.svelte.ts`, including delete of session-minted clones), Reset all, and Save and apply. Clicking "New theme" opens a preset picker (Sunrise, Graphite, Sepia, Nordic, High-contrast Dark, Lavender Pastel; "Start blank" keeps the seed-from-active path). The editor body renders every group inline with no mode selector. The editor surfaces an inline rebake banner whenever a loaded theme's `derivationEngineVersion` trails the code constant `DERIVATION_ENGINE_VERSION` and no dismissal row exists for the (theme, current version) pair: clicking Rebake re-runs derivation for non-isolated tokens, stamps the new version, and updates `blendCanvas` if `--cal-bg` is non-isolated, all in the buffer until commit; clicking Maybe later records the dismissal in memory and writes the `theme_upgrade_dismissals` row immediately for an existing theme, or queues it in `pendingDismissals` so it lands right after the parent insert when a fresh theme is committed. The `.dark` class on the HTML root tracks the actual canvas luminance (`isThemeDark` / `isThemeCalendarDark`) and event-tile palette selection consumes the same luminance check; the sun/moon icon is now a separate, purely decorative `iconLabel` tag (`light` or `dark`) that user themes can flip with one click in the editor header (built-ins peg `iconLabel` to their `base`). The flip is persisted on the theme (DB columns `themes.icon_label` / `themes.seed_icon_label`, originally added as `scheme`/`seed_scheme` in migration v5 and renamed in migration v7; JSON v1 export carries an `iconLabel` field with tolerant fallback to the older `scheme` key on import; legacy imports and rows with NULL columns default from canvas luminance via `defaultIconLabelFromCanvas` and are backfilled on hydrate). Migration v6 also drops the now-redundant `base` column from `themes`: user themes paint directly from their stored token snapshot, only `BuiltinTheme` keeps a `base` discriminator, and the legacy JSON import path falls back to `defaultIconLabelFromCanvas` to pick a BASE table when it needs to derive missing tokens. Every pair row shows a live contrast indicator with per-pair targets (4.5:1 body, 3:1 recessed) and an auto-fix wand, a persistent contrast-summary bar reports total failing pairs with Jump-to-next and Fix-all, the in-house HSL color picker drives every editable field and an editable JSON panel that commits through `replaceTheme` (id locked), and JSON import/export works through both clipboard and native file dialog.
